📌 **프로젝트명**

QA 자동화 테스트 도입 및 학습 프로젝트

## 1. 배경
**기존 QA 업무**에서 반복적인 테스트(로그인, 검색 등)를 매번 수동으로 수행하면서  
**시간 소모와 테스트 누락** 가능성을 느꼈습니다.

**Regression Test** 과정에서 동일 시나리오를 반복해야 하는 비효율을 개선하고자  
**자동화 테스트**를 학습하고 적용해보는 프로젝트 진행

## 2. 문제 정의
❌ 동일 테스트 반복 → 시간 낭비

❌ 수동 테스트 → 휴먼 에러 발생

❌ 테스트 결과 → 기록/추적 불가


## 3. 접근 방식
Selenium/pytest 학습하며 **웹 자동화 테스트** 구현했습니다.

**학습 포인트:**
CSS Selector, XPATH → 요소 선택 원리

pytest 구조 → 테스트 실행 흐름

HTML 리포트 → 결과 시각화


## 4. 구현 내용
[네이버 검색 테스트]

✅ 검색어 입력 → 결과 페이지 검증

✅ 빈 검색어 → 동작 확인

✅ 블로그 탭 → 이동 검증

[Tapas 로그인 테스트]

✅ 정상 로그인 → 메인 페이지 진입

✅ 로그인 실패 → 에러 메시지 검증

[시나리오 테스트]

✅ 검색 → 블로그 → 첫 게시글 클릭


## 5. CI 적용
**GitHub Actions**로 **매일 오후 2시 자동 실행** 환경 구축

✅ pytest-html 리포트 자동 생성

✅ Actions 탭 → report.zip 확인

✅ CI 개념 학습 중 (구조 이해 중)

## 6. 결과
- 반복 테스트 자동화를 통해 동일 시나리오 수행 시간 단축
- 테스트 결과를 리포트로 확인 가능하여 추적성 개선
- CI 기반 자동 실행으로 테스트 누락 방지 환경 구성
